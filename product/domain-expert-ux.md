# 领域专家用户体验设计

## 文档概述

本文档定义 Univers 平台的产品设计和用户体验，专注于**领域专家**这一核心用户群体。

**设计理念**：
- 让专家专注于**元层思考**（"什么是好问题"），而非执行（"怎么解决"）
- **低摩擦**捕获知识，在工作流中自然沉淀
- **主动提醒**被忽视的尺度，辅助多尺度思维
- **看得见的价值**，让专家知道自己的贡献如何创造价值

**非目标**：
- ❌ 不是"AI 工具"（我们不卖工具）
- ❌ 不是"自动化软件"（我们不替代人，而是放大人）
- ❌ 不是"知识管理系统"（我们不是文档库，而是知识执行平台）

## 一、用户画像

### 1.1 核心用户：领域专家

#### 用户 A：张师傅（老师傅）

```yaml
基本信息:
  角色: HVAC 资深技师
  经验: 20 年现场经验
  年龄: 45 岁
  技术能力: 中等（会用手机和电脑，但不精通）

日常工作:
  - 巡检设备（每天 6-8 小时）
  - 排查故障（凭经验判断）
  - 指导年轻技师
  - 填写工作日志（纸质或 Excel）

痛点:
  - 80% 时间在做重复性工作（巡检、记录）
  - 宝贵的经验只在脑子里，年轻人学不到
  - 想提建议，但没时间整理、没渠道表达
  - 知道很多"诀窍"，但说不清楚"为什么"

期望:
  - 释放时间，做更有价值的事（思考、改进）
  - 经验能被记录、被认可、能变现
  - 简单易用，不要复杂培训
```

```yaml
基本信息:

  - 数据驱动决策，而非拍脑袋
  - 团队知识共享，降低对个人依赖
```

## 二、核心工作流

### 2.1 工作流 1：日常运维（释放时间）

#### 场景
张师傅每天需要巡检 50 台设备，记录参数、排查异常。

#### 传统方式
```
1. 手动巡检每台设备（8 小时）
   - 看参数是否正常
   - 听声音是否异常
   - 检查外观是否有问题

2. 手写记录在本子上（30 分钟）

3. 回办公室录入 Excel（30 分钟）

4. 如果发现问题，拍照、写报告（1 小时）

总时间：10 小时
思考时间：0 小时
```

#### Univers 方式
```
1. AI 自动监控 50 台设备（24/7）
   - 实时采集参数
   - 检测异常模式
   - 自动记录日志

2. 推送异常到张师傅手机（即时）
   "设备 #23 发现异常：启动电流波动 6%，声音频率偏移 3Hz"

3. 张师傅判断（5 分钟）
   - 看监控数据
   - 远程听声音（如果有传感器）
   - 判断："是，这是轴承磨损早期征兆"

4. 系统记录张师傅的判断（自动）
   - 情境：电流波动 + 声音异常 + 温度正常
   - 诊断：轴承磨损
   - 行动：2 周内更换
   - 来源：张师傅

5. 张师傅确认行动（1 分钟）
   - "生成工单，安排更换"

总时间：6 分钟（主动判断）
思考时间：5 分钟
释放出来的时间：9.9 小时
```

#### 产品设计要点

**1. 移动优先**
```
张师傅主要在现场工作，设计移动端为主：
- 推送通知：异常即时推送
- 语音输入：快速记录判断
- 拍照上传：快速附加证据
```

**2. 极简交互**
```
通知示例：
┌────────────────────────────────┐
│ 🚨 设备异常                     │
│                                │
│ 设备 #23 (A 栋压缩机)           │
│ 异常：电流波动 6%, 声音偏移 3Hz │
│                                │
│ [查看详情] [忽略] [记录判断]    │
└────────────────────────────────┘

点击 [记录判断] 后：
┌────────────────────────────────┐
│ 这是什么问题？                  │
│                                │
│ 🎤 点击说话                    │
│                                │
│ 或选择常见问题：                │
│ [ ] 轴承磨损                    │
│ [ ] 制冷剂泄漏                  │
│ [ ] 电路故障                    │
│ [ ] 其他                        │
└────────────────────────────────┘
```

**3. AI 辅助，人类决策**
```
AI 提示："这个模式与张师傅之前诊断的'轴承磨损'相似度 87%"
→ 人类确认或修正
→ 系统学习
```

### 2.2 工作流 2：知识捕获（沉淀经验）

#### 场景
张师傅在工作中发现了一个"诀窍"，系统如何低摩擦地记录下来？

#### 触发时机

**时机 1：专家主动输入**
```
张师傅在处理问题时，点击"记录经验"按钮
```

**时机 2：AI 检测到元层思考**
```
系统检测到张师傅说了：
"这个问题看起来是 X，但其实根本原因是 Y"

AI 主动提示：
┌────────────────────────────────┐
│ 💡 检测到重要洞察               │
│                                │
│ 你刚才的观点可能帮到其他人！    │
│ 要花 2 分钟记录下来吗？         │
│                                │
│ [是的] [稍后] [不用]            │
└────────────────────────────────┘
```

**时机 3：模式识别**
```
系统发现：张师傅连续 3 次在类似情境下做了相同判断

AI 提示：
"你似乎总在 X 情况下判断为 Y，这是一个规律吗？
要不要总结成一条经验？"
```

**3. 社交激励**
- "李工点赞了你的经验"
- "你的经验本月被使用 8 次"

### 2.3 工作流 3：多尺度决策（元层思考）

#### 场景
李工需要决策："是否投入资源开发故障预测功能？"

#### 传统方式
```
1. 李工凭直觉判断："故障预测很重要，应该做"
2. 召集团队讨论，意见不一
3. 拍板决定，开始开发
4. 3 个月后发现：客户不买单（因为定价太高）
5. 反思：也许应该先验证客户意愿支付价格
```

#### Univers 方式

**步骤 1：初始决策草案**
```
李工输入：
"我觉得应该投入 3 个月开发故障预测功能，
 理由是客户需要，技术上可行。"
```

**步骤 2：缩放提示引擎介入**
```
系统分析：
- 当前尺度：技术可行性（细节层面）
- 被忽视的尺度：
  * 商业可行性（客户付费意愿）
  * 长期战略（这是否符合公司方向）
  * 团队能力（是否有人力）

AI 提示：
┌────────────────────────────────┐
│ 💡 也许可以从其他尺度考虑       │
│                                │
│ 当前思考：技术层面（是否能做）  │
│                                │
│ 其他视角：                      │
│                                │
│ 🔹 商业视角：客户愿意为故障预测│
│    支付多少？定价能否覆盖成本？ │
│                                │
│ 🔹 战略视角：这符合我们"长期   │
│    聚焦能效"的战略吗？还是说   │
│    我们要转向"故障管理"？      │
│                                │
│ 🔹 执行视角：团队有 3 个月的   │
│    人力吗？这会影响其他项目吗？│
│                                │
│ [查看更多视角] [继续当前决策]  │
└────────────────────────────────┘
```

**步骤 3：切换尺度，重新思考**
```
李工选择"商业视角"，系统展示：

┌────────────────────────────────┐
│ 商业视角：客户付费意愿          │
│                                │
│ 数据（来自平台洞察）：          │
│                                │

│   - 74% 认为故障预测重要        │

│                                │
│ 💡 建议：先做 MVP 验证，而非   │
│    完整开发                     │
│                                │
│ [查看详细分析] [返回]           │
└────────────────────────────────┘
```

**步骤 4：元层洞察推荐**
```
系统检索知识库，发现相关洞察：

┌────────────────────────────────┐
│ 📚 相关洞察                     │
│                                │
│ 洞察："HVAC 客户更在乎零故障，  │
│       而非故障预测"             │

│ • 不要卖"预测软件"              │
│ • 而是卖"零故障保障服务"        │
│ • 定价模式：按 SLA 收费         │
│                                │
│ [采纳此洞察] [了解更多]         │
└────────────────────────────────┘
```

**步骤 5：综合决策**
```
李工重新审视决策：

原计划：
"开发故障预测功能（3 个月，$90K）"

调整后：
"做 MVP 验证（1 个月，$30K）：
 1. 先找 5 个客户，测试付费意愿

 3. 验证'零故障 SLA'模式是否可行

┌────────────────────────────────┐

│ 考虑的尺度：                    │

│ ✓ 商业可行性（新增）            │
│ ✓ 战略对齐（新增）              │
│ ✓ 资源约束（新增）              │
│                                │
│ 应用的洞察：                    │
│ • "客户更在乎零故障，而非预测" │
│                                │
│ 预期效果：                      │
│ • 风险降低 70%（MVP vs 全量）   │
│ • 成本降低 67% ($30K vs $90K)  │
│ • 验证周期缩短 66% (1月 vs 3月)│
│                                │
│ [确认执行] [继续调整]           │
└────────────────────────────────┘
```

#### 产品设计要点

**1. 非侵入式提醒**
- 不要打断用户流程
- 以"建议"的方式提示，而非"阻止"
- 用户可选择"继续当前决策"

**2. 数据驱动**
- 提供具体数字（不是空泛建议）
- 来源清晰（"来自 200 个客户调研"）
- 可追溯（点击可查看原始数据）

**3. 可视化尺度**
```
当前决策的尺度雷达图：

      战略 (20%)
          |
技术 (90%) ●------ 商业 (30%)
          |
      执行 (40%)

点击任意维度，展开该视角的分析
```

## 四、关键体验原则

### 4.1 低摩擦原则

**定义**：每个操作的摩擦力 < 10 秒

**实践**：
```
❌ 不好的设计：
"请填写 15 个字段，详细描述你的经验..."
→ 用户不会填

✅ 好的设计：
"🎤 点击说一句话"
→ 5 秒完成
```

**具体策略**：
1. **语音优先**：现场工作，语音比打字快 10 倍
2. **AI 填充**：用户说一句话，AI 自动补全其他字段
3. **分步引导**：不要一次展示大表单
4. **智能默认值**：基于历史数据预填

### 4.2 即时反馈原则

**定义**：每个操作都有即时、具体的反馈

**实践**：
```
❌ 不好的设计：
"你的经验已保存"（然后呢？）

✅ 好的设计：
"你的经验已保存！
 • 预计帮助 23 个项目
 • 类似经验平均创造 $3,000 价值
 • 你将获得 30% 分成"
```

### 4.3 主动辅助原则

**定义**：AI 主动帮助，但不打扰

**实践**：
```
时机：用户在思考决策时
方式：温和提示，而非强制
文案："💡 也许可以从其他角度考虑？"（而非"你必须考虑 X"）
选择：用户可以选择"不用，继续"
```

### 4.4 可见价值原则

**定义**：让用户清楚看到"我的贡献如何创造价值"

**实践**：
```
每周报告：
┌────────────────────────────────┐
│ 本周你的贡献                    │
│                                │
│ 📝 记录经验：3 条               │
│ 💡 提出洞察：1 条               │
│ ✅ 处理异常：12 次              │
│                                │
│ 🌍 你的影响力                   │
│ • 你的经验被使用 15 次          │
│ • 帮助 8 个项目                 │
│ • 节省成本 $12,000              │
│                                │
│ 💰 你的收益                     │
│ • 本周：+$1,500                 │
│ • 本月：+$5,200                 │
│ • 累计：$45,000                 │
│                                │
│ 🏆 排名：本领域 Top 5%          │
│                                │
│ [查看详细] [分享成就]           │

```

## 八、总结

### 核心设计理念
1. **释放时间**：让 AI 做执行，专家做思考
2. **低摩擦沉淀**：在工作流中自然捕获知识
3. **多尺度辅助**：主动提醒被忽视的尺度
4. **可见价值**：贡献者看到影响力和收益

### 差异化优势
- **不是工具**，是"外部大脑"
- **不是软件**，是"知识网络"
- **不是自动化**，是"智能放大"
