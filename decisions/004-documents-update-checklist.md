## 更新原则

### 关键用词调整

**避免使用**：
- ❌ "平台化"（暗示 SaaS）
- ❌ "订阅模式"
- ❌ "对外销售 Workbench"
- ❌ "客户使用我们的平台"
- ❌ "SaaS 产品"

**应该使用**：
- ✅ "服务"、"专业服务"
- ✅ "按价值/结果收费"
- ✅ "Workbench 是内部工具"
- ✅ "客户购买结果"
- ✅ "AI 驱动的服务公司"

### 核心信息

在更新时，应该强调：

1. **商业模式**：
   ```
   Univers = AI 驱动的 HVAC 优化服务公司
   不是：SaaS 平台公司
   ```

2. **Workbench 定位**：
   ```
   主要（90%）：内部运营平台
   次要（10%）：专家社区工具（可选）
   ```

3. **收费模式**：
   ```
   主要：按价值/结果收费
   - 能源节省分成
   - 项目固定费
   不是：订阅费
   ```

4. **客户体验**：
   ```
   客户买：HVAC 优化服务（结果）
   客户做：提需求、验收结果
   客户不做：学习使用平台、管理 AI
   ```

5. **类比**：
   ```
   ✅ 类似：Uber（服务公司）、McKinsey（咨询）
   ❌ 不像：Salesforce（SaaS）、AWS（平台）
   ```

## 特别说明

### 关于"平台"一词

**不是禁止使用"平台"**，而是需要明确：

✅ 可以说：
- "Workbench 是我们的内部运营平台"
- "用平台规模化服务"
- "元层知识平台"（如果指的是知识网络）

❌ 不应该说：
- "Workbench 是我们的对外产品"
- "客户使用我们的平台"
- "平台订阅收费"

### 关于"服务 vs 平台"

我们的定位：
```
客户视角：服务公司（买结果）
内部视角：用平台工具规模化服务
类比：Uber

不是：
客户视角：平台公司（买软件）
内部视角：卖软件订阅
类比：Salesforce
```
